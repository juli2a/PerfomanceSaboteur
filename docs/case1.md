# Case 1: Оптимізація зображень

**Категорія:** Rendering / Медіа-оптимізація
**Тумблер:** Rendering → Image Optimization
**Метрика:** LCP

## Короткий опис
Вплив відсутності сучасних форматів стиснення та відкладеного завантаження картинок на метрику LCP (Largest Contentful Paint).

---

## Гарний код (Тумблер OFF)

**Реалізація:**
```tsx
<Image
  src={product.thumbnail}
  alt={product.title}
  width={80}
  height={80}
  sizes="(max-width: 768px) 40px, 80px"
  className="object-cover"
/>
```

**Поведінка інтерфейсу:**
- Зображення вантажаться у форматі WebP/AVIF, вага до 5–10 КБ.
- Метрика LCP горить зеленим (значення до 1.2s).
- Картинки з'являються плавно по мірі скролу.

---

## Поганий код (Тумблер ON)

**Реалізація:**
```tsx
<img
  src={product.images[0]}
  alt={product.title}
  className="w-[80px] h-[80px] object-cover"
/>
```
> Запит тягне оригінальне, нестиснене важке зображення з масиву `images`.

**Поведінка інтерфейсу:**
- Браузер завантажує оригінальні файли по кілька мегабайт для кожного рядка/картки.
- Метрика LCP виходить у червону зону (понад 4.0s), особливо на мобільному.
- Панель відображає: `"LCP: 4.8s (⚠️ Critical - Unoptimized Images)"`.

---

## Аналіз

**Ймовірність успішної демонстрації: 8/10**
DummyJSON повертає `thumbnail` (стиснений) та `images[]` (більші оригінали з CDN). Next.js `<Image>` автоматично конвертує в WebP та lazy-loads. Різниця буде відчутна. Єдиний ризик — якщо DummyJSON CDN почне повертати вже оптимізовані файли, різниця зменшиться.

**Відповідність UI та кейсу:** ✅ Повна. Inventory Control показує колонку "Мініатюра" у таблиці та квадратне зображення у картках — саме там і відбувається заміна `<Image>` ↔ `<img>`.

**API:** ✅ `product.thumbnail` та `product.images[]` — обидва поля існують у кожному об'єкті DummyJSON `/products`.
