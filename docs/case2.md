# Case 2: Зсув макету при завантаженні

**Категорія:** Rendering / Візуальна стабільність
**Тумблер:** Rendering → Layout Shift
**Метрика:** CLS

## Короткий опис
Відсутність резервування простору під важкі динамічні блоки призводить до стрибків інтерфейсу та псує метрику CLS (Cumulative Layout Shift).

---

## Гарний код (Тумблер OFF)

**Реалізація:**
```tsx
// Дані передаються як пропси з Server Component
// SalesChart підключається через dynamic({ ssr: false })
<div className="h-[400px] w-full bg-muted animate-pulse rounded-xl">
  <SalesChart data={chartData} />
</div>
```
> Фіксована висота контейнера → скелетон видимий поки recharts монтується → геометрія сторінки не змінюється.

**Поведінка інтерфейсу:**
- Сторінка завантажується стабільно, геометрія не змінюється.
- Метрика CLS = `0.0`.
- Контент під графіком не рухається.

---

## Поганий код (Тумблер ON)

**Реалізація:**
```tsx
// Контейнер без фіксованої висоти
<div className="h-auto w-full">
  <SalesChart data={chartData} />
</div>
```
> До монтування recharts — висота контейнера = 0px. Дані вже є (передані з сервера), але Chart ще не ініціалізований.

**Поведінка інтерфейсу:**
- Користувач бачить верхні картки та блоки під графіком.
- Коли дані надходять, графік різко розгортається на 400px вниз, зштовхуючи нижню частину сторінки.
- Метрика CLS спалахує червоним: `"CLS: 0.35 (⚠️ Bad - Layout Shift Detected)"`.

---

## Аналіз

**Ймовірність успішної демонстрації: 9/10**

`recharts` — суто клієнтська бібліотека, підключається через `dynamic(() => import('./Chart'), { ssr: false })`. У server-rendered HTML місце під Chart **порожнє** (висота 0px). Коли JS-бандл завантажується і recharts ініціалізується, Chart монтується і розгортається на ~400px → зсуває контент нижче → CLS відбувається **природньо, без штучних затримок**.

**Реалізація:** дані фетчаться у Server Component і передаються як пропси до клієнтського Chart. Жодного client-side fetch не потрібно. Toggle контролює лише клас обгортки:
- OFF: `h-100 w-full` → фіксований контейнер → скелетон поки Chart монтується → CLS = 0
- ON: `h-auto w-full` → висота 0 до монтування → розширення на 400px → CLS

На повільних з'єднаннях і реальних девайсах ефект ще виразніший через довший час завантаження бандлу recharts.

**Відповідність UI та кейсу:** ✅ "Головний графік продажів" на Dashboard — опис повністю відповідає.

**API:** ✅ Дані для графіка з `/carts?limit=100` існують. Клієнтський fetch не потрібен.
